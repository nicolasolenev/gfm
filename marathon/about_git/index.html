Небольшой справочник про Git

git init — создает папку .git в корневой дирректории проекта;

git status — проверяет наличие неотслеживаемых файлов;

git add <имя файла> — добавляет файл в индекс;

git commit — запоминает состояние файлов из индекса (создает коммит, далее можно будет вернуться к этому состоянию в любое время);

git commit --amend — добавляет изменения в предыдущий коммит (используется в случае, если что-то забыли добавить в индекс перед коммитом, а также отредактировать комментарий к коммиту);

git checkout -b <имя ветки> — создает новую ветку проекта и переходит в нее (изменения в этой ветке не затронут основную ветку, далее можно будет изменения перенести в основную ветку);

git log — посмотреть историю коммитов;

git restore <имя файла> — позволяет вернуться к исходному состоянию, если изменения не были закоммичены;

git branch — позволяет посмотреть, какие ветки есть в проекте;

git checkout <имя ветки> — позволяет переместиться в другую ветку проекта;

git merge <имя ветки> — позволяет перенести изменения из одной ветки в другую (например, вы находитель с ветке master, команда git merge features перенесет изменения из ветки features в ветку master;
